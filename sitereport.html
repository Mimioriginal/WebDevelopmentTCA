Starting with HTML and CSS

15/APR/2022

Learning Experience and Debugging Process

At the beginning of the module, the first step was to identify the best text editor to use for coding the website. I have never created a website before so I experimented with different tools such as Atom, Sublime and Visual Studio Code.

I found that Visual Studio Code was the best option because it has the right features and is the right fit for my first website. It was really easy to link it to Github and had many other easy to use features such as creating new branches, merging branches and committing new updates.

Chrome's "inspect" option helped me a great deal with debugging. I was able to select certain elements, see grid layout lines, grid area names, the positions of elements and use this information to identify any code that was not working as expected. It also allowed me to adjust the height and width of the page to ensure it was working correctly on different sized screens.

What Went Well

Figuring out a simple way to create the animated border on the hover effect in my nav menu was a fast and easy method of making the buttons aesthetically pleasing. At first I thought that it would take me days to figure out how to create keyframe animations, however I was able to realise my idea within about 10 minutes of research.

The grid layout allowed me to easily position the html elements within the website, making the pages look organised.

Using CSS to control the style and layout of the pages enabled me to have a consistently designed website without the need to format each element individually by utilising classes and ID's.

What Went Wrong

When first trying to utilise classes within the CSS file, I attempted to write .name_of_class in my CSS file however Visual Studio Code wasn't suggesting to autofill the class from my html file. Unfortunately due to my lack of experience with the program at this time I did not realise that it only auto fills once you have used the class before. I did originally attempt to type it out however the web page did not look as expected because the actual issue was not with VSC, it was a problem with the code itself. This set me back many hours of trying to identify the issue before realising what the problem was.

Another issue I had during the project was the layout of the navigation bar. It was intended to have all the buttons aligned in one row however as the width of the page reduced, the end buttons would start moving into a new row within their grid cell. This was corrected by using responsive width and height measurement units that allowed the navigation bar to move left on smaller screens.

When designing the overall layout of my website using grid, I kept forgetting to write display: grid; or to list the 'grid-template-area's which caused some content on the page to be misaligned.

Technical

I created the index.html page with the correct DOCTYPE, head, title and body (main and footer). I then added the nav tag with an unordered list and links to the relevant pages that I needed in my nav bar. A great feature within Visual Studio Code editor is the ability to CTRL+click on the html links listed inside the nav, which allowed me to create a new html file as well as the autofill suggestions as this saves a lot of time.

Using the grid feature to dictate the layout of the web site allowed me to easily adjust where each HTML element should be displayed. Having the ability to divide each page into rows and columns to customise individual cells meant that each one had a custom name, size and position.

The navigation bar was created using a grid layout to ensure it was consistently aligned across all pages. It includes features such as changing colour on hover for desktops and on tap for mobile. This was achieved by using the ":hover" and ":active" effect.

By utilising classes, I was able to style the currently open page using by highlighting that specific nav bar link with a different colour.

Designing the UI

When deciding the design for my website, I looked at other web developers websites such as: Steve Wolf, Saltik, Thaipham Photography.

After conducting research on different websites such as the ones listed above, I noticed that some of them have too many animations that make the pages look overly clustered and unprofessional. I came to the conclusion that my navigation bar should be positioned in the top right for ease of access with the logo and my name situated on the top left, and a footer at the bottom of the page (as per layout.drawio in Github). This is common in most commercial websites which means it will be intuitive for users visiting mine.

One thing I knew from the start was that I wanted a dark themed website. They are great because they put less strain on my eyes. When I visit a web site, the first thing I usually look for is the dark theme button.

Since I opted for dark background colours, I needed bright coloured fonts. I love the combination of purple with the dark theme as it makes the titles stand out and the bright text elevates the dark background theme making the website more colourful.
