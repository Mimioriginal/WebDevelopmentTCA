Starting with HTML and CSS

15/APR/2022

Learning Experience and Debugging Process

Chrome's inspect option helped me a great deal with debugging. I was able to select certain elements, see grid layout lines, grid area names, the positions of elements. It also helped me figure out which widths and heights I needed to use.

What Went Well

My favourite moment was when I figured out a simple way to create the animated border on hover effect in my nav menu. At first I thought that it would take me days to figure out how to create keyframe animations, however I was able to realise my idea within about 10 minutes of research.

What Went Wrong

My least favorite moment was when I thought that Visual Studio Code transfers the classes to the CSS file from my html files. I tried to write .name_of_class in my CSS file however Visual Studio Code wasn't suggesting to autofill the class from my html file. It baffled me. For 4 hours. That was one miserable afternoon. I felt like I wasted a lot of valuable time. I had tried to type out the class anyway and apply the styles that I wanted, however my website still didn't work. The truth was that I just didn't understand how the grid layout worked at this time. However in reality, I was looking for an answer to the wrong question: "How to fix my text editor?"

Technical

First I created the index.html page with the correct DOCTYPE, head, title, body (main and footer). I then added the nav tag with an unordered list and links to the relevant pages that I will be needing in my nav bar. One think I love about Visual Studio Code editor is that I can CTRL+click on the html links listed inside the nav and it will allow me to create a new html file as well as the autofill suggestions as this has saved me a lot of time.

Learning about columns took the least amount of time for me. Especially in comparison to the grid layout, which took me about 4 days to master. With the grid layout I kept forgetting either to write display: grid; or to list the 'grid-template-area's. However with columns I was able to create a layout within about 5 minutes of reading about them. Although I must admit that at first, when I realised that the portfolio needs to be in columns, I thought that it will take me the same amount of time as the grid layout.

Designing the UI

Something that helped me decide on my website design, was looking at other web developers portfolios. A lot of them have tons of animations, especially on the homepage, however I'm not really a fan of that. I understand that it's a great way of showcasing skills however I believe that less is sometimes more. A nice clean professional look is what I chose. One thing I knew from the start was that I wanted a dark theme website. They are great because they put less strain on my eyes. When I visit a new web page, the first thing I usually look for is the dark theme button.

Since I opted for dark background colours, I needed bright coloured fonts. I love the combination of purple with the dark theme as it makes the titles pop out a bit. Also the bright text elevates the dark backgroud theme, making the website a bit more cheerful.
